java中的集合类Collection

我的理解

网上找的图

————————————————

List , Set, Map都是接口，前两个继承至Collection接口，Map为独立接口
Set下有HashSet，LinkedHashSet，TreeSet
List下有ArrayList，Vector，LinkedList
Map下有Hashtable，LinkedHashMap，HashMap，TreeMap
Collection接口下还有个Queue接口，有PriorityQueue类
————————————————

— List 有序,可重复

ArrayList
优点: 底层数据结构是数组，查询快，增删慢。
缺点: 线程不安全，效率高
Vector
优点: 底层数据结构是数组，查询快，增删慢。
缺点: 线程安全，效率低
LinkedList
优点: 底层数据结构是链表，查询慢，增删快。
缺点: 线程不安全，效率高
————————————————

—Set 无序,唯一

HashSet

底层数据结构是哈希表。(无序,唯一)
如何来保证元素唯一性?
1.依赖两个方法：hashCode()和equals()

HashSet 在调用add()方法时，会先调用equals(Object o)来比较新项和已有的某项是否相等，若不相等时，会调用hashCode()得到对象的散列码，根据这个码计算出对象的存储位置。

LinkedHashSet
底层数据结构是链表和哈希表。(FIFO插入有序,唯一)
1.由链表保证元素有序
2.由哈希表保证元素唯一

TreeSet

底层数据结构是红黑树。(唯一，有序)

  1. 如何保证元素排序的呢?
    自然排序
    比较器排序
  2. 如何保证元素唯一性的呢?
    根据比较的返回值是否是0来决定

迭代器Iterator

Iterator是java.lang包中自带的接口，被Collection继承。

Iterator只能向后遍历，不能向前遍历。

Iterator常用方法

iterator()：要求容器返回一个Iterator对象

next()：第一次调用时返回第一个元素，以后调用发那会下一个元素。

hasNext()：检查序列中是否还有元素。

remove()：将返回的元素删除。

List list;
Iterator iterator = list.iterator();
while(iterator.hasNext())
    {
    System.out.print(iterator.next();)
    }

迭代器ListIterator

继承于Iterator，被List继承，Set不能用。可以双向遍历，可以插入和删除元素

next();

previous(); 向前遍历

hasNext();

hasPrevious(); 判断是否有前一个

add(); 添加一个元素

nextIndex(); 前一个元素的索引值，即下标

previousIndex(); 后一个元素的索引值，即下标

*set(); * 修改当前元素的值

List list = new ArrayList();

ListIterator li = list.listIterator();
while(li.hasNext())//+迭代输出
while(li.hasPrevious())//+迭代输出
